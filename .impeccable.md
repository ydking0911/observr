# observr Design Context

## Design Context

### Users
개발자 (indie dev ~ 소규모 팀). 로컬 또는 on-prem 환경에서 자신이 만든 서비스의 HTTP 트레이스와 로그를 실시간으로 확인하는 용도. AI agent(Claude Code, Cursor 등)가 사용하는 CLI와는 분리된 사람용 인터페이스. 개발 중 "지금 어떤 요청이 들어오고 있나?", "어디서 에러가 났나?"를 빠르게 파악하는 것이 핵심 job.

### Brand Personality
**정밀한 · 간결한 · 신뢰가는**

- 데이터를 정확하게 보여주는 도구. 화려함보다 명확함.
- 지표가 한눈에 읽혀야 하고, 에러는 즉시 눈에 띄어야 함.
- 불필요한 장식 없이 정보 자체가 중심.

### Aesthetic Direction
- **레퍼런스**: Toss (toss.im) — 깔끔한 카드 레이아웃, 절제된 타이포그래피, 신뢰감 있는 색상, 여백의 미
- **테마**: 라이트 모드 기반. 순백이 아닌 따뜻한 오프화이트 배경.
- **색상**: 주조색은 딥 네이비/차콜 계열, 에러는 명확한 레드, 경고는 앰버, 정상은 그린. 악센트 컬러 한 가지만 사용 (Toss 블루 계열 — `oklch(55% 0.18 250)`).
- **폰트**: `Pretendard` (한국 개발자 친화적, Toss가 사용) 또는 `DM Sans` + `JetBrains Mono` (수치/코드).
- **안티 레퍼런스**: Datadog의 복잡한 sidebar, Grafana의 어두운 네온 색상, 유리 형태소 효과, 사이버펑크 느낌.

### Design Principles
1. **지표 우선**: 숫자와 상태가 가장 먼저 눈에 들어와야 한다. 설명 텍스트는 최소화.
2. **에러는 즉시 인식**: 에러 레벨 이벤트는 색상, 무게, 위치 모두에서 명확히 구분된다.
3. **여백이 말한다**: 항목 간 간격이 곧 관계의 표현. 카드 중첩 없이 평탄한 계층 구조.
4. **모노크롬 + 단일 액센트**: 회색 스펙트럼 위에 Toss 블루 한 가지만 액센트로 사용.
5. **실시간이지만 차분하게**: WebSocket 실시간 업데이트도 깜빡임/플래시 없이 부드럽게 삽입.
